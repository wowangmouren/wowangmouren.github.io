四、接口

inferface 一种能力，约束（约束对象的属性值的类型）

1.可选属性： sex ? : string

2.只读属性：readonly id : number

3.函数类型：接口也可描述函数了欸行

4.类类型：

​ （1）类实现接口：implements

​ 一个类实现多个接口： class Cars implements 接口1,接口2

​ （2）接口继承接口：extends

五、类

基于类的面向对象的方式，单例模式也可以理解为是class类的实例，仅有这一个实例

1.继承： extends : 派生类（子类）从基类（超类）中继承属性和方法

super: 子类调用父类的方法或构造函数时使用，子类也可重写父类的方法

2.类成员的修饰符

​ （1）private:私有的，只有类内部可以访问

​ （2）protected：受保护的，类内部和子类中可以访问

​ （3）public： 公开的，默认值，外部可以访问

​ （4）readonly ： 只读

​ （5）参数属性：相当于类的简写，

3.存取器：类属性的获取（get函数）、类属性的改写（set函数）

4.类的静态属性： static

5.抽象类：抽象类其子类可以实例化（extend继承后再实例）

六、函数：匿名函数、命名函数

1.可选参数（？）和默认参数（未传参时为undefind）

2.剩余参数（…args:string）

3.函数重载：函数名相同，而参数不同的多个函数

七、泛型

在定义函数、接口、类时未预定类型 ，也可以写成其他字母

1.函数泛型

2.多个泛型参数的函数：<T，V>,使用逗号隔开

3.泛型接口

4.泛型约束：泛型参数直接取属性值会报错，必须先继承接口

八、其他

1.声明文件：使用第三方库时，需先引用它的声明文件

2.内置对象：可以在TS中当定义好了的类型使用

​ Number、Boolean、String、Date、RegExp、Error
