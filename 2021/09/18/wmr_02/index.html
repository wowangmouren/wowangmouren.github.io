git使用

September 18, 2021 660

一、协同开发，最好是考虑下需要合并的情况：

  • git stash save “save message” : 执行存储时，添加备注，方便查找，只有git stash 也要可以的，但查找时不方便识别。
  • git pull ：没有改动的情况下，可以去掉（1）跟（3）直接拉取代码。
  • git stash pop ：命令恢复之前缓存的工作目录，将缓存堆栈中的对应stash删除
  • 编译器上手动解决代码冲突问题
  • git add .
  • git commit -m “提交说明”
  • git push

二、常用命令：

  • git status ：查看状态
  • git commit -m “提交说明”
  • git reset HEAD 文件名: 撤销add
  • git reset –soft HEAD^: 撤回commit跟add的提交，返回上一版本
  • git checkout – 文件: 撤销没add修改
  • git log 查看记录
  • git diff 文件路径 对比文件改动
  • gitk 对比文件改动
  • git commit -s -m “说明”
  • git push origin dev 提交
  • 使用前设置用户信息：
    git config –global user.name ‘runoob’
    git config –global user.email test@runoob.com
